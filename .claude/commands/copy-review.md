---
description: Review all user-facing copy in a file or component against the Granted brand voice
---

Using the brand-voice agent rules, audit the file or component specified (or the last file edited if none given).

Find every string a user will read:
- Button labels
- Headings and subheadings
- Empty states
- Error messages
- Placeholder text
- Toast/notification messages
- Loading states

For each one, evaluate:
1. Does it lead with what the user gets (not what the feature does)?
2. Is it free of filler and corporate jargon?
3. If it's an error, does it tell the user what to do next?
4. If it's an empty state, does it invite action?
5. Does it avoid "AI-powered" or similar marketing language?

Output a table:

| Current copy | Issue | Suggested revision |
|---|---|---|

Then apply the fixes directly to the file if there are clear improvements. Flag anything that needs a product decision before changing.
